# アプリ要件定義書 (Requirements Definition)

## 1. プロジェクト概要
*   **アプリの目的**: 飲食店やスーパーなどで利用可能な決済手段をマップ上で可視化し、ユーザー同士の協力で情報を充実させるアプリ。
*   **アプリ名**: PayMap
*   **対応プラットフォーム**: iOS専用

## 2. システム・技術スタック
*   **フロントエンド**: Swift / SwiftUI
*   **アーキテクチャ**: MVVM
*   **バックエンド / データベース**: Firebase (無料枠の範囲内で利用: Firestore, Auth, Storage)
*   **地図API**: Google Maps SDK for iOS / Google Places API (店舗情報の初期データ取得用)
*   **課金実装**: StoreKit / RevenueCat

## 3. コア機能の仕様 (Spec)

### 3.1. マップ＆店舗表示機能
*   アプリ起動後、現在地周辺のGoogleマップを表示。
*   店舗データは**Google Places API**を用いて動的に取得・表示する。
*   店舗アイコン（ピン）をタップすると、その店舗で利用可能な決済手段が表示される。

### 3.2. 決済手段の登録・情報の正確性担保
*   ユーザーが任意の店舗に対して利用可能な決済手段を登録・追加できる。
*   **情報の自浄作用（コンセンサス方式）**: 複数ユーザーの投稿（例：3人以上が「PayPayが使える」と投稿）が一致した場合に、確定情報としてマップに反映・採用する方式とする。

### 3.3. ユーザー認証とプロフィール
*   **認証方法**: Appleでサインイン、および Googleログイン (Firebase Auth)

### 3.4. インセンティブシステム（ゲーミフィケーション）
*   **バッジ（称号）機能**: 特定の条件（登録件数など）を満たすと称号を獲得できる。
*   **貢献度ランキング**: ユーザー間の登録数を月間/総合などでランキング表示する。
*   *[追加アイデア候補]*:
    *   **ファーストディスカバリー・ボーナス**: その店舗の決済手段を「初めて」登録したユーザーには特別高いポイントを付与する。
    *   **チェックイン（確認）ボーナス**: 既に登録されている情報に対し、「今もこの決済が使えました」と確認ボタンを押すだけで少量の貢献ポイントがもらえる仕組み。

### 3.5. 有料機能（プレミアム会員）
*   **広告非表示**: 無料版では広告を表示し、有料版で非表示に。
*   **高度な絞り込み検索**: 「PayPayが使えて、深夜営業しているカフェ」などのクロス検索。
*   *[追加アイデア候補]*:
    *   **マイリスト（お気に入りリスト）の作成と共有**: 「Suicaが使えるお気に入りカフェリスト」などを作成できる機能。
    *   **オフラインマップ機能**: 電波の悪い地下のお店などに備え、特定のエリアのマップと決済情報をダウンロードしておける機能。
    *   **オリジナルマップピン**: マップ上に表示される現在地のアイコンや、自分が登録した店舗のピンを特別なデザインに変更できる着せ替え機能。
